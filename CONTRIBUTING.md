# Contributing

Thanks for your interest in Agent Identity Layer.

## What we want in early contributions

At this stage, the project is documentation-first. The highest-value contributions are:

- clarifying the problem statement
- improving schema boundaries
- adding realistic interoperability examples
- sharpening security and trust assumptions
- identifying ambiguous terms or overclaims

## What to avoid for now

Please avoid opening PRs that immediately add:

- heavyweight crypto frameworks without clear deployment need
- blockchain or tokenized identity concepts
- speculative compliance language
- provider-specific lock-in disguised as a standard
- large implementation scaffolds before the core model is stable

## Contribution workflow

1. Open an issue or discussion for significant changes
2. Keep proposals narrow and concrete
3. Prefer examples and threat analysis over hype
4. Submit a PR with a clear explanation of what changed and why

## Style guidelines

- Use plain English
- Prefer practical language over standards theater
- Distinguish clearly between current scope and future work
- Keep examples realistic and auditable
- Do not include secrets, credentials, or private user data in examples

## PR checklist

- [ ] The change improves clarity, interoperability, or safety
- [ ] Claims are not stronger than the current evidence supports
- [ ] New examples do not include sensitive data
- [ ] Terminology is consistent with the README and spec
- [ ] Non-goals are still respected
